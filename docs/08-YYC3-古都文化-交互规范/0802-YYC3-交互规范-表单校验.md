# 表单验证与反馈

## 一、P5 旨意工坊表单校验规则

| 字段 | 规则 | 校验时机 | 错误提示 |
|------|------|---------|---------|
| 旨意标题 | 必填，2-50 字，不可仅空白 | 失焦 | 朱砂红竖线 + "请拟旨意标题" |
| 紧急程度 | 必选一项 | 提交 | "请选择旨意紧急程度" |
| 指派六部 | 至少选 1 个部门 | 提交 | "请至少指派一个执行部门" |
| 旨意详情 | 必填，10-2000 字 | 失焦 | "旨意详情不可少于 10 字" |
| 圣旨类型 | 必选 | 默认"敕书" | — |

## 二、校验状态视觉

| 状态 | 边框 | 图标 |
|------|------|------|
| Default | `1px solid rgba(212,168,67,0.12)` | — |
| Valid | `1px solid #10b981`（碧色） | 右侧 ✓ 翠涛 |
| Error | `1px solid #c2414b`（朱砂红） | 右侧 ✕ 朱砂 + 提示文字 |
| Focus | `1px solid #d4a843`（鎏金） + 微光晕 | — |

## 三、提交按钮状态机

```
Idle → Validating → Submitting → Success → Reset
  │        │            │           │
  └────────┴────────────┴───────────┘
           Error (shake + red flash)
```

| 状态 | 按钮文字 | 视觉 | 可点击 |
|------|---------|------|:------:|
| Idle | "奉天承运 · 颁旨" | 鎏金渐变 | 是 |
| Validating | "奉天承运 · 颁旨" | 微光脉冲 | 否 |
| Submitting | "颁旨中..." | 圣旨卷轴发光 + 旋转龙纹 | 否 |
| Success | "✓ 旨意已颁" | 碧色 + 玉玺印记 | 否（2s 后重置） |
| Error | "颁旨失败 · 重试" | 红色 + 抖动 | 是 |

## 四、存为草稿

- 按钮：Secondary ghost 样式，位于 [颁旨] 左侧
- 存储：localStorage `edict_draft_{timestamp}`
- 草稿列表入口：P6 旨意板顶部 Tab "草稿箱"
- 草稿卡片：灰底 + "草稿" 标签 + 最后编辑时间 + 删除按钮

## 五、自动保存与草稿恢复

| 触发时机 | 策略 |
|---------|------|
| 内容变更后空闲 3s | debounce 自动保存 |
| 每 30s 定时 | 心跳保存 |
| 切换字段焦点 | 即时保存 |
| beforeunload | 最终保存 |
| visibilitychange | 紧急保存 |

**恢复流程：** 进入 P5 → 检测草稿 → 显示恢复提示 → [恢复草稿] / [新建旨意] / [删除草稿]
