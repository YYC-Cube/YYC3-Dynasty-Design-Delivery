# 全局状态管理

## 一、全局状态 Store 定义

| Store | 职责 | 关键数据 | 持久化 |
|-------|------|---------|:------:|
| `edictStore` | 旨意数据 | 列表、详情、流转状态 | session |
| `ministryStore` | 六部健康 | 令牌数、健康状态、日志 | — |
| `personaStore` | AI人格 | 12人格列表、当前选中 | local |
| `dynastyStore` | 朝代数据 | 13朝代、Skill列表 | — |
| `notificationStore` | 通知系统 | 通知队列、未读数 | session |

## 二、全局加载状态

| 层级 | 触发 | 表现 |
|------|------|------|
| 页面级 | 路由切换 / 首次数据加载 | Skeleton 骨架屏 |
| 组件级 | 局部数据刷新 | 组件内 Loading 态 |
| 操作级 | 提交/保存/催办 | 按钮内旋转龙纹 |
| 下拉级 | 无限滚动加载 | 底部 → 旋转加载器 |

## 三、通知系统 · 朝钟飞鸽烽火

| 级别 | 古称 | 触发场景 | 音效 | 显示时长 | UI 表现 |
|:----:|:----:|---------|:----:|:-------:|---------|
| P0 | 烽火·紧急 | 令牌耗尽、系统异常 | 铜磬 800Hz | 手动关闭 | 红色横幅 + 即时插队 |
| P1 | 朝钟·重要 | 封驳/驳回/系统告警 | 编钟 523Hz | 8s | 琥珀色 + 队列首位 |
| P2 | 飞鸽·常规 | 任务完成/阶段变更 | 无 | 5s | 蓝色 + 正常队列 |
| P3 | 笺·提示 | 保存成功/复制成功 | 无 | 3s | 灰色 + 末尾队列 |

**通知驱动链：**
```
服务器推送 → Notification Store → 导航栏角标更新
                                → Toast 弹出（优先级队列）
                                → 通知中心列表更新
                                → 音效触发（P0/P1）
```
