# 从 Figma 到代码的交接清单

## 一、设计交付包结构

```
YYC3-Dynasty-Design-Delivery/
├── figma-file/         YYC3-Dynasty-v1.0.fig
├── assets/             icons/ / illustrations/ / seals/ / patterns/ / fonts/
├── tokens/             colors.json / typography.json / spacing.json / shadows.json / radii.json
├── components/         6 个核心组件 Spec (DynastyButton / EdictStepBar / MinistryCard / SealChain / DynastyNavbar / SearchPalette)
├── pages/              11 个页面 Spec (P1-P11)
├── animations/         seal-stamping.json / scroll-unfurl.json / dragon-glow.json / cloud-drift.json
└── README.md           交接说明
```

## 二、Component Properties 完整定义

| 组件 | Variant | Boolean | Text | Instance Swap |
|------|:-------:|:-------:|:----:|:------------:|
| EdictStepBar | Stage Count / Stage Status | Show Timestamps | — | Current Stage |
| DynastyButton | State (5种) | Has Skill Badge | Dynasty / Skill Count | Icon |
| MinistryCard | Ministry / Health Status | Show Metrics / Compact | — | 六部图标 |
| SealChain | Seal Count / Direction | Show Labels | — | Seal 1-7 |
| DynastyNavbar | Active Nav Item | Show Breadcrumb / Search | Notification Count | — |
| SearchPalette | Mode (4种) | Show Categories | Result Count | — |

## 三、Instance Swap 映射

- MinistryCard ↔ 六部图标（6个）
- 天子驾六 ↔ 马状态（Normal/Warning/Error）
- 玉玺 ↔ 印章状态（Idle/Hover/Pressing/Stamped/Locked）
- 签章链 ↔ 签章节点（Signed/Pending/Vetoed）

## 四、导出命名规范

```
格式: {Section}_{Component}_{Variant}_{Size}@{Scale}x.{Format}

示例: Navigation_DynastyNavbar_Default_1440@2x.png
      Icon_Seal_Stamped_80@1x.svg
```
