# 三省六部权责定义

## 概述

三省六部制是 YYC³ Dynasty 的核心治理架构。**三省分权制衡，六部专责执行**，严格遵循唐代三省六部制的历史逻辑，映射为系统角色与数据流转路径。

---

## 一、三省：决策制衡层

### 中书省（凤阁）— 草拟

| 属性 | 值 |
|------|-----|
| 古称 | 凤阁 |
| 色彩 | 石青 `#3b82f6` |
| 系统角色 | 方案生成器 |
| 核心职责 | 旨意草拟、方案库匹配、格式规范检查 |
| 权限等级 | P2 |

**系统映射：**
- 接收太子分拣后的旨意需求
- 从方案库匹配最佳模板
- 生成旨意草拟稿（白麻形态）
- 上传至门下省待审议队列

**UI 标识：**
- 石青色标识色
- 图标：毛笔 + 竹简
- 位置：P6 旨意板"草拟中"阶段标记、P2 应天门左阙

---

### 门下省（鸾台）— 审议

| 属性 | 值 |
|------|-----|
| 古称 | 鸾台 |
| 色彩 | 藤黄 `#f59e0b` |
| 系统角色 | 合规检查器 |
| 核心职责 | 典章合规检查、封驳、签章审批 |
| 权限等级 | P2 |

**系统映射：**
- 按"律令格式"四元体系校验旨意合规性
- 通过则签章（进入尚书省派发队列）
- 不通过则封驳（退回中书省重拟，附带驳回理由）
- 封驳时显示朱砂批红 + 驳回理由面板

**UI 标识：**
- 藤黄色标识色
- 图标：天平 + 法典
- 位置：P6 "审议中"阶段标记、P2 应天门中阙

---

### 尚书省（文昌府）— 派发

| 属性 | 值 |
|------|-----|
| 古称 | 文昌府 |
| 色彩 | 翠涛 `#10b981` |
| 系统角色 | 调度分配器 |
| 核心职责 | 令牌配额管理、部门调度、派发执行 |
| 权限等级 | P2 |

**系统映射：**
- 检查对应六部令牌配额是否充足
- 分配鱼符令牌 → 派发至对应部门
- 跟踪执行进度
- 办结后缴回鱼符、归档

**UI 标识：**
- 翠涛色标识色
- 图标：令牌 + 地图
- 位置：P6 "派发中"阶段标记、P2 应天门右阙

## 三、权力流转拓扑

```
天子（天堂）
  │ 降旨
  ▼
太子（明堂）
  │ 分拣
  ▼
中书省（应天门左阙） → 草拟 → 白麻形态
  │
  ▼
门下省（应天门中阙） → 审议 → 通过则绯绫，封驳则退回
  │
  ▼
尚书省（应天门右阙） → 派发 → 下发鱼符令牌
  │
  ▼
六部（天津桥） → 执行 → 消耗令牌配额
  │
  ▼
归档（定鼎门） → 闭环 → 缴回鱼符
```
