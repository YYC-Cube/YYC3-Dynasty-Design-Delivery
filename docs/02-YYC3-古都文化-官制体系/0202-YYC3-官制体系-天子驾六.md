# 天子驾六：六马状态机与六部健康映射

## 概述

"天子驾六"为周代最高礼制——天子车驾由六匹马牵引。在 YYC³ Dynasty 系统中，**六马对应六部，每匹马的健康状态代表对应部门的运行状态**。

---

## 一、六马与六部映射

| 马名 | 毛色 | 对应部门 | 健康指标 | UI 位置 |
|------|------|---------|---------|---------|
| 骅骝 | 赤马（鎏金） | 户部 | 数据调用成功率、算力使用率 | P2 左上 |
| 騄駬 | 白马（霜白） | 吏部 | Agent在线率、考核完成率 | P2 右上 |
| 骊驹 | 黑马（朱砂） | 兵部 | 部署成功率、Bug修复率 | P2 左中 |
| 騑騑 | 灰马（墨灰） | 刑部 | 安全扫描通过率、审计覆盖率 | P2 右中 |
| 骐骥 | 棕马（古铜） | 工部 | CI/CD成功率、构建通过率 | P2 左下 |
| 騋牝 | 紫马（绀紫） | 礼部 | 文档生成成功率、规范覆盖率 | P2 右下 |

## 二、健康状态定义

每匹马有三种健康态，对应 P10 太史监候的实时监控数据：

| 状态 | 视觉 | 含义 | 触发条件 |
|------|------|------|---------|
| ✅ Normal | 马身发光，姿态昂扬 | 正常运行 | 所有指标 > 80% |
| ⚠️ Warning | 马身琥珀色光晕，微颤 | 负载偏高 | 任一指标 60-80% |
| ❌ Error | 马身红色脉冲，垂头 | 运行异常 | 任一指标 < 60% |

## 三、六马悬浮动效参数

每匹马独立动画，营造"御风而行"的自然感：

| 参数 | 值 |
|------|-----|
| Float Y | ±6px（垂直悬浮） |
| Duration | 3-5s（每匹马周期不同） |
| Easing | Ease In and Out |
| Stagger Delay | 每匹马 0.3s |
| 排列顺序 | 骅骝→騄駬→骊驹→騑騑→骐骥→騋牝 |

## 四、健康状态切换动效

| 切换方向 | 时长 | 动效描述 |
|---------|------|---------|
| Normal → Warning | 300ms | 马身颜色渐变（琥珀化），马蹄微颤 |
| Warning → Error | 500ms × 2次 | 马身红色脉冲，绳索闪烁 |
| Error → Normal | 800ms | 马身颜色恢复，金色粒子修复 |
| 全局刷新 | 600ms | 六马同时重置，stagger 0.1s |

## 五、状态机流转规则

```
Normal ──(指标下降)──→ Warning ──(指标继续下降)──→ Error
  ↑                        │                            │
  └──────(指标恢复)────────┘────────(人工修复)───────────┘
```

- 状态只能逐级变化（不可 Normal ↔ Error 跳跃）
- 人工修复（P5 旨意派发修复任务）可使 Error → Normal
- 自动修复仅支持 Warning → Normal
- 状态变化触发通知：Normal 不通知 / Warning 飞鸽通知 / Error 烽火告警
