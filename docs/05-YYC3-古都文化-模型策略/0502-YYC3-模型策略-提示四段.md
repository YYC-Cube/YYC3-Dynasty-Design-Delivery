# Figma Make 提示词四阶段策略

## 第一阶段：初始提示词（The Brief）

**适用模型**：Claude Opus 或 Gemini 3.1 Pro
**原则**：首条提示词必须完整、意图明确，包含目标、上下文、关键元素、行为约束、完成标准。

### 提示词结构模板

```
提示词结构模板：
1. 角色定位（你是一位XX专家）
2. 页面目标（请为我创建XX页面）
3. 尺寸规格（Frame: 1440×900）
4. 布局描述（分区 + 元素位置）
5. 设计系统引用（色值/字体/间距）
6. 动效描述（触发时机 + 时长 + 缓动）
7. 输出要求（Component + Variant + Auto Layout）
```

### 示例（P1 启动页）

```
You are a senior UI designer specializing in ancient Chinese imperial aesthetics
fused with modern dark-mode dashboards. Create a welcome/launch page for
"YYC³ Dynasty" — an AI multi-agent governance platform styled after the
Tang Dynasty's "Three Departments and Six Ministries" system.

FRAME: 520×680px, dark theme.

LAYOUT:
1. TOP: 56×56px jade seal icon, title "YYC³ Dynasty" 28px gold, subtitle
2. MIDDLE: 3×2 grid of entry cards with icon + title + description
3. BOTTOM: Golden divider, status text, CTA button "奉天承运"

COLORS: bg #0a0e17, gold #d4a843, card bg rgba(212,168,67,0.08)
... [完整提示词见 06-页面提示]
```

## 第二阶段：增量迭代（The Delta）

**适用模型**：Claude Sonnet 4.6 或 Gemini 3 Flash
**原则**：每次只改一类事物，明确"改什么/怎么改/不变什么"。

### 提示词结构模板

```
提示词结构模板：
1. 目标组件/区域（Update the XX component）
2. 具体变更（Change the background from A to B）
3. 保持不变（Keep all other properties unchanged）
4. 状态覆盖（Apply to all variants: default/hover/active/disabled）
```

### 示例

```
Update the DynastyButton component in P3.
- Change: hover background from rgba(212,168,67,0.08) to rgba(212,168,67,0.15)
- Change: add border glow on active state (box-shadow: 0 0 20px rgba(212,168,67,0.4))
- Keep: all other properties unchanged (size, font, spacing, border-radius)
- Apply to: all 13 dynasty button variants
```

## 第三阶段：多页一致性校验（Consistency Check）

**适用模型**：Claude Opus
**原则**：逐页校验 Design Token 一致性、组件引用一致性、动效参数一致性。

### 校验清单模板

```
Consistency check across all 11 pages:

1. Color Token 一致性:
   - P1-P11 的 --color-bg-primary 是否均为 #0a0e17？
   - 所有按钮的 accent 是否为 #d4a843？

2. 组件引用一致性:
   - P5 和 P9 的圣旨组件是否引用同一个 EdictScroll 组件？
   - P6 和 P9 的 EdictStepBar 是否同一组件 variant？

3. 动效参数一致性:
   - 所有 Toast 通知是否统一使用 400ms ease-out？
   - 页面切换是否统一 400ms ease-in-out？
```

## 第四阶段：最终交付（Final Polish）

**适用模型**：Gemini 3.1 Pro（创意润色）+ Claude Sonnet 4.6（结构化校验）

### 润色内容

```
Final polish checklist:

1. 组件状态补全: 每个组件是否覆盖了 Idle/Hover/Active/Disabled 四态？
2. 响应式检查: 1440px 布局在 1024px 是否合理折叠？
3. 文案校对: 所有古风用语是否准确？有无错别字？
4. 视觉层级: 信息层级是否清晰？focus point 是否正确？
5. 文化准确度: 龙纹/云纹/回纹等是否遵循规范？
```
