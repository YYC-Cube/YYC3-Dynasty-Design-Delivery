# 五模型能力画像与选型矩阵

## 五模型能力画像

| 模型 | Figma 官方定位 | 推理特征 | Credits 消耗 | 最佳适用场景 |
|------|---------------|---------|-------------|-------------|
| **Gemini 3 Flash** | Fast, iterative | 快速响应、擅长简单迭代、低延迟生成 | 低 | 组件微调、文案修改、单页快速迭代 |
| **Claude Sonnet 4.6** | Balanced, efficient | 均衡推理、结构化输出稳定、JSON/组件结构可靠 | 中 | 标准页面生成、组件库搭建、Design Token 绑定 |
| **Gemini 3.1 Pro** | Deep, creative | 长上下文理解、创意发散、多模态融合 | 中高 | 全页创意设计、文化符号融合、复杂动效描述 |
| **Claude Opus** | Thorough, uses more credits | 深度推理、视觉保真度高、多步骤逻辑链完整 | 高 | 复杂交互闭环、多页一致性校验、动效分镜设计 |
| **GPT-5.5** | — (Figma Make 支持) | 复杂布局处理、编码能力强、系统级思维 | 高 | 全局架构设计、Design Token 体系、响应式断点 |

## 模型选型决策矩阵

按 YYC³ Dynasty 项目 11 页面 + 组件库的复杂度，推荐采用**分层混合模型策略**：

```
第一层「架构筑基」→ Claude Opus / GPT-5.5
  ├─ 全站 Design Token 体系
  ├─ 11 页面信息架构与导航系统
  └─ 组件库 Variants 规划

第二层「核心页面」→ Gemini 3.1 Pro / Claude Opus
  ├─ P2 朝堂大厅（中轴线 + 五层叠加）
  ├─ P5 旨意工坊（圣旨动效 + 玉玺盖章）
  └─ P8 双星桥（双架构桥接动效）

第三层「标准页面」→ Claude Sonnet 4.6
  ├─ P1 欢迎启动页 / P3 十三王朝 / P4 勋章墙
  ├─ P6 旨意板 / P7 Hub 浮窗
  └─ 基础组件库

第四层「迭代微调」→ Gemini 3 Flash
  ├─ 文案调整、颜色微调
  ├─ 单组件状态补全
  └─ 响应式适配
```

## 模型选择流程图

```
任务类型？
├─ 全站架构 / Design Token → Claude Opus / GPT-5.5
├─ 核心创意页面           → Gemini 3.1 Pro / Claude Opus
├─ 标准页面 / 组件库       → Claude Sonnet 4.6
└─ 细节微调 / 迭代        → Gemini 3 Flash

Credits 预算？
├─ 充裕 → 优先 Claude Opus
├─ 适中 → Claude Sonnet 4.6
└─ 紧张 → Gemini 3 Flash + 手动调整
```

## 模型切换策略

- **初始生成**：使用 Gemini 3.1 Pro（创意发散能力最强）
- **结构化稳定**：转为 Claude Sonnet 4.6（结构化输出最可靠）
- **一致性校验**：使用 Claude Opus（多页对比最全面）
- **最终润色**：Gemini 3.1 Pro（创意润色）+ Claude Sonnet 4.6（结构化校验）
