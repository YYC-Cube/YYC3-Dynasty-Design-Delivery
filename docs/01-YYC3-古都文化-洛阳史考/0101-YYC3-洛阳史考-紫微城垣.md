# 洛阳紫微城中轴线空间映射

## 概述

所有 Agent 节点均对应洛阳紫微城真实地标，**空间方位 = 权力层级 = 数据流转方向**。从北到南依次：天堂 → 明堂 → 应天门（中书/门下/尚书） → 天津桥（六部） → 端门（早朝） → 定鼎门（归档）。

## 中轴线全景

```
        北（权力源头）
        │
   ┌────┴────┐  ── 天堂（天子）       → 降旨发令 / 全局最终决策 / 不可驳回
   │         │
   │  明堂   │  ── 明堂（储君）       → 旨意分拣 / 格式标准化 / 监国代行
   │         │
   ├─────────┤  ── 应天门（三省）     → 草拟·审议·派发 三权分立
   │中 门 尚 │    左阙中书: 待草拟 / 方案库匹配
   │         │    中阙门下: 待审议 / 典章合规检查
   ├──┬──┬──┤    右阙尚书: 待派发 / 令牌配额
   │户│礼│吏│
   │ 天津桥 │  ── 天津桥（六部）     → 令牌驱动执行 / 资源消费
   │兵│刑│工│
   ├──┴──┴──┤
   │  端门   │  ── 端门（早朝）       → 定时播报 / 健康巡检 / 风险预警
   │         │
   │ 定鼎门  │  ── 定鼎门（归档）     → 任务闭环 / 数据汇总 / 历史回溯
   │         │
   └────┬────┘
        │
        南（执行终端）
```

## 节点详表

| 节点 | 地标 | 系统角色 | 核心职责 | 数据流向 |
|------|------|---------|---------|---------|
| 天堂 | 洛阳天堂（通天浮屠） | 天子/全局最高决策 | 降旨发令、最终签章、系统熔断 | 北→南 单向 |
| 明堂 | 洛阳明堂（万象神宫） | 太子/监国 | 旨意分拣、格式标准化、代行批阅 | 北→南 过滤分发 |
| 应天门·左阙 | 应天门（中书省） | 中书省/草拟 | 方案匹配、旨意草拟、格式规范 | 北→南 待草拟流 |
| 应天门·中阙 | 应天门（门下省） | 门下省/审议 | 典章合规检查、封驳、签章 | 中阙截流转 |
| 应天门·右阙 | 应天门（尚书省） | 尚书省/派发 | 令牌配额、部门调度、派发 | 南→北 入栈 |
| 天津桥 | 洛阳天津桥 | 六部/执行 | 令牌驱动、资源消费、任务执行 | 水平六部并行 |
| 端门 | 洛阳端门 | 早朝/巡检 | 定时播报、健康巡检、风险预警 | 全局广播 |
| 定鼎门 | 洛阳定鼎门 | 归档/历史 | 任务闭环、数据汇总、历史回溯 | 南端终结 |

## 空间映射原则

| 原则 | 说明 |
|------|------|
| **北高南低** | 权力源头在北（天子），执行终端在南（六部） |
| **三权分立** | 应天门三阙对应草拟/审议/派发，不可跨阙操作 |
| **中轴为纲** | 所有核心节点沿中轴线排列，东西对称（六部分列天津桥两侧） |
| **不可逆流** | 数据严格从北向南流转，仅封驳可逆流回北 |
