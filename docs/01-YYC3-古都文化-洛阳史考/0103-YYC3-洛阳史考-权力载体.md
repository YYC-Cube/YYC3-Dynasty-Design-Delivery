# 权力载体：文化符号 → 系统底层功能映射

## 概述

四类古物分别对应系统的**权限、消息、规则、凭证**四大底层能力：

```
传国玉玺  →  最高权限体系    →  鉴权/签章/熔断
圣旨卷轴  →  全链路消息载体  →  创建/流转/归档
竹箴典章  →  规则与知识库    →  合规/审议/稽核
鱼符令牌  →  执行权限与凭证  →  配额/资源/认证
```

---

## 6.1 传国玉玺 → 最高权限体系

| 玉玺名称 | 权限等级 | 系统功能 | 触发节点 |
|----------|---------|---------|----------|
| 传国玉玺 | P0 最高级 | 全局最终签章、系统级配置修改、熔断重置 | 天堂 |
| 皇帝行玺 | P1 政务级 | 日常行政任务生效 | 天堂 |
| 皇帝信玺 | P2 军事级 | 紧急任务/Bug修复授权 | 天堂/紧急敕牒 |
| 皇帝之玺 | P3 人事级 | Agent注册/权限变更/考核奖惩 | 天堂/吏部 |
| 天子之玺 | P4 对外级 | 第三方对接/跨域资源申请 | 天堂/工部 |

**玉玺使用规则：**
- 五玺不可同时使用，同一时间仅一玺有效
- 传国玉玺可覆盖任何下级玉玺的操作
- 玉玺使用记录全部记入 P10 太史监候审计日志
- 玉玺锁定态：Token 过期/权限不足时自动降级

**UI 表现：**
- 高权限操作前显示玉玺选择面板
- 当前可用玉玺高亮，不可用灰白锁定
- 盖章时显示对应玉玺名称 + 九叠篆印文

## 6.2 圣旨卷轴 → 全链路消息载体

| 圣旨类型 | 系统消息类型 | 适用场景 | 流转路径 |
|----------|-------------|---------|----------|
| 制书 | `edict_sys` | 全局制度变更、架构调整（最高等级） | 皇帝→太子→三省→六部（全流程） |
| 敕书 | `edict_norm` | 日常任务、常规需求（最常用） | 皇帝→太子→中书→门下→尚书→六部 |
| 敕牒 | `edict_fast` | 快速指令、临时调整、紧急修复 | 太子→中书→尚书→六部（门下事后补审） |
| 堂帖 | `edict_dept` | 尚书省内部调度、六部协同 | 尚书省→六部之间内部流转 |
| 密奏/封事 | `memorial_secret` | 异常上报、风险预警，直达皇帝 | 六部/门下→直达天堂 |

**圣旨四阶形态（流转中自动变化）：**

```
白麻（初稿）→ 黄纸（中书草拟后）→ 绯绫（门下通过后）→ 金泥（玉玺钤印后·正式生效）
```

| 阶段 | 形态 | 视觉特征 | 可操作 |
|------|------|---------|--------|
| 1-下旨 | — | 圣旨收起，轴头可见 | 仅天子可见 |
| 2-分拣 | — | 圣旨半展，标签分类 | 太子操作 |
| 3-草拟 | 白麻 | 素白麻纸底，无装饰 | 中书省编辑 |
| 4-审议 | 黄纸 | 淡黄纸底 + 云纹镶边 | 门下省审批 |
| 5-派发 | 绯绫 | 淡红绫底 + 金线镶边 | 尚书省调度 |
| 6-执行 | 金泥 | 明黄底 + 鎏金镶边 + 玉玺印 | 六部执行 |
| 7-归档 | 金泥（已封） | 明黄底 + 封条 + 归档章 | 只读 |

## 6.3 竹箴典章 → 规则与知识库

对应隋唐"律令格式"四元法律体系，为门下省审议唯一依据：

| 分类 | 对应规则 | 示例 | UI 表现 |
|------|---------|------|---------|
| 律 | 系统红线规则、安全禁令、硬性约束 | 不可绕过审批、不可越权操作 | 红色警示标签 |
| 令 | 部门职能规范、权责边界、流程标准 | 中书不得越权审议 | 蓝色指引标签 |
| 格 | 奖惩细则、考核标准、权限配额 | 连续3次封驳该部门扣分 | 琥珀色度量标签 |
| 式 | 操作规范、输出格式、接口标准 | API返回格式、日志记录规范 | 灰色规格标签 |

**竹箴的 UI 呈现：**
- 弹窗形式："典章参照"面板（⌘+D 唤起）
- 搜索结果中高亮匹配的典章条款
- 审议时自动弹出关联的律令格式条目

## 6.4 鱼符令牌 → 执行权限与资源凭证

| 令牌 | 持有部门 | 对应功能 | Token 名称 |
|------|---------|---------|-----------|
| 户部鱼符 | 户部 | 数据调用权限、算力配额、存储资源 | `token:hubu` |
| 礼部鱼符 | 礼部 | 规范发布、文档生成、知识库编辑 | `token:libu` |
| 兵部鱼符 | 兵部 | 代码仓库读写、部署权限、Bug修复 | `token:bingbu` |
| 刑部鱼符 | 刑部 | 安全扫描、审计权限、风险处置 | `token:xingbu` |
| 工部鱼符 | 工部 | CI/CD流水线、服务器操作、工具链配置 | `token:gongbu` |
| 吏部鱼符 | 吏部 | Agent管理、权限分配、考核奖惩 | `token:libu_hr` |

**核心规则：**
- 尚书省派发任务时同步下发鱼符
- 任务办结后缴回注销
- 鱼符过期自动失效（时效：按旨意紧急程度设定）
- 鱼符可被天子/太子手动吊销

**鱼符 UI：**
- 尺寸：32×16px（横排鱼形）
- 颜色：对应六部色彩
- 状态：有效（发光）/ 已缴回（灰白）/ 已吊销（红色斜杠）
- 位置：P2 六部卡片右上角、P6 旨意卡片右下角
